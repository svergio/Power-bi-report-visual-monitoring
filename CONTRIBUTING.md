# Руководство для участников

## Подготовка среды

1. Создайте виртуальное окружение:
   - `python -m venv .venv`
   - `. .venv/bin/activate` (Linux/macOS) или активируйте `.venv` в Windows
2. Установите зависимости:
   - `pip install -r requirements.txt`
3. Локальная конфигурация:
   - `cp .env.example .env`
4. Запуск тестов:
   - `pytest`

Для `python -m pbimonitor` задайте `PYTHONPATH=src` (см. README).

## Стиль кода

- Делайте изменения небольшими и сфокусированными.
- Не коммитьте секреты, внутренние URL, учётные данные и файл `.env`.
- Комментарии — по минимуму, только для нетривиальной бизнес-логики.
- Предпочитайте понятные имена вместо длинных пояснений в комментариях.

## Чеклист pull request

- [ ] Для изменений поведения добавлены или обновлены тесты
- [ ] Проходит `pytest`
- [ ] Проходит `flake8`
- [ ] Проходит `mypy src tests`
- [ ] Обновлены README или другая документация при смене поведения или конфигурации
- [ ] В коде, примерах и документации нет чувствительных данных

## Изменения в конфигурации отчётов

При добавлении примеров отчётов:

- используйте URL вида `https://powerbi.example.com/...`;
- стабильные идентификаторы в `snake_case`;
- укажите поля `interval`, `threshold`, `enabled`.

## Сообщения об ошибках

Укажите:

- какую команду монитора использовали;
- релевантные ключи из `.env` (без секретов);
- журналы и трассировку стека;
- ожидаемое и фактическое поведение.
